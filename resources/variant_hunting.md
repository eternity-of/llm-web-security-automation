# Variant Hunting Guide

하나의 취약점이 확인되면 동일 원인이 적용된 다른 기능을 찾아야 한다.

## 1. 공통 확장 질문

- 같은 파라미터 이름을 사용하는 다른 요청이 있는가?
- 같은 객체 ID를 사용하는 조회/수정/삭제/다운로드가 있는가?
- 같은 PHP/JSP/Controller 네이밍 패턴이 있는가?
- 같은 응답 메시지 또는 JSON 구조를 사용하는 기능이 있는가?
- 같은 권한 경계가 다른 기능에도 적용되는가?
- 같은 선행/후속 단계 구조가 반복되는가?

## 2. 인증 누락 발견 시

추가 확인 대상:

- 동일 기능군의 목록/상세/다운로드/이미지 API
- 세션 제거 요청
- 로그아웃 후 재사용 요청
- 만료 세션 요청

확장 질문:

- 인증이 필요한 화면과 실제 API 검증이 일치하는가?
- HTML 페이지는 막히지만 파일/API는 열리는가?
- 비로그인 상태에서 민감 데이터가 반환되는가?

## 3. IDOR / 인가 미흡 발견 시

추가 확인 대상:

- 조회
- 수정
- 삭제
- 다운로드
- 취소
- 승인
- 이력 조회
- 알림 조회

확장 질문:

- 같은 객체 ID가 다른 요청에도 쓰이는가?
- 목록에 없는 객체를 직접 접근할 수 있는가?
- A 계정 객체를 B 계정 세션에서 조작할 수 있는가?
- 관리자 객체와 일반 사용자 객체가 같은 식별자 체계를 쓰는가?

## 4. 파일 다운로드 인증/인가 누락 발견 시

추가 확인 대상:

- down, download, file, attach, image, view_file, editor upload/download
- fileId, fileNo, attachId, number, path, filename 파라미터

확장 질문:

- 다운로드가 게시글/문서 권한과 연결되어 있는가?
- 비공개 게시글의 첨부파일도 직접 다운로드 가능한가?
- 인증 제거 후 파일이 반환되는가?
- 순차 ID로 접근 범위를 넓힐 수 있는가?

## 5. CSRF 발견 시

추가 확인 대상:

- write, update, delete, cancel, approve, change, reset, upload, reserve
- GET 기반 상태 변경 요청
- CSRF Token 없는 POST 요청
- Origin/Referer 미검증 요청

확장 질문:

- 모든 상태 변경 요청에 토큰이 있는가?
- 토큰을 제거/변경해도 성공하는가?
- SameSite 쿠키 속성이 있는가?
- Stored XSS 또는 HTML Injection과 결합 가능한가?

## 6. Stored HTML Injection / XSS 발견 시

추가 확인 대상:

- 목록 페이지
- 상세 페이지
- 관리자 페이지
- 검색 결과
- 알림
- 메일
- PDF/엑셀 다운로드
- 같은 입력값이 재출력되는 모든 위치

확장 질문:

- 출력 위치별 인코딩이 다른가?
- 관리자 화면에서 같은 값이 재출력되는가?
- HTML은 허용되지만 이벤트/스크립트가 차단되는가?
- 저장형 입력값이 다른 사용자에게 노출되는가?

## 7. SQL Injection 징후 발견 시

추가 확인 대상:

- 검색
- 상세 조회
- 로그인
- 정렬/필터
- 날짜 조건
- 통계/리포트
- 엑셀 다운로드

확장 질문:

- 같은 파라미터가 다른 요청에도 쓰이는가?
- 오류 메시지 패턴이 반복되는가?
- 문자열/숫자 타입 변경에 응답 차이가 있는가?
- 정렬 파라미터가 허용 목록 검증을 거치는가?

## 8. 계정 복구/비밀번호 재설정 문제 발견 시

추가 확인 대상:

- 이메일 변경
- 휴대폰 인증
- OTP 등록/해제
- 관리자 비밀번호 초기화
- 회원가입 인증
- 초대 링크
- 인증번호 재발급

확장 질문:

- 후속 단계에서 선행 검증 결과를 재검증하는가?
- 실패한 선행 단계가 세션 상태를 오염시키는가?
- 임시 토큰이 일회성/만료/계정 바인딩을 갖는가?
- 직접 후속 URL 호출이 가능한가?

## 9. Variant 결과 표시

각 취약점 상세에는 다음을 포함한다.

```text
유사 패턴 확인:
- 확인한 유사 기능:
- 동일 원인 가능성:
- 실제 확인 여부:
- 추가 확인 필요:
```
